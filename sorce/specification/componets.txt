План Storybook: Компоненты FSE
Цель: Организовать и отслеживать разработку всех UI-компонентов проекта. Этот документ служит дорожной картой для frontend-команды.

1. Готовые компоненты из shadcn/ui
Эти компоненты считаются готовыми после установки и базовой стилизации под нашу дизайн-систему (цвета, шрифты). Они составляют основу нашего UI-кита.

Атомарные компоненты:
[x] Button (Кнопка)

[x] Input (Поле ввода)

[x] Select (Выпадающий список)

[x] Checkbox

[x] Badge (Тег / Pill)

[x] Avatar (Аватар)

[x] Tooltip (Всплывающая подсказка)

[x] Separator (Разделитель)

Молекулярные компоненты:
[x] Accordion (Используется для Filter Group)

[x] Dropdown Menu (Используется для User Menu)

[x] Pagination (Пагинация)

[x] Dialog (Модальное окно)

[x] Toast (Уведомление)

2. Компоненты для сборки / генерации в V0
Это наши кастомные, уникальные компоненты. Они требуют отдельной разработки. Мы можем использовать V0 для генерации их первоначальной структуры, а затем дорабатывать вручную.

Приоритет 1: Ключевые компоненты для MVP
[ ] Work Card (Карточка работы)

Задача: Собрать из Card, Badge и текстовых элементов. Это самый важный кастомный компонент.

Метод: Идеальный кандидат для V0. Промпт: "Create a responsive card component for a story using shadcn/ui Card. It should display a title, author, a list of fandom badges, a summary, and a footer with stats icons for rating, word count, and likes."

[ ] Search Bar (Строка поиска)

Задача: Скомбинировать Input и Button с иконкой поиска из lucide-react.

Метод: Ручная сборка, занимает 15-20 минут.

[ ] Reader Controls (Управление читалкой)

Задача: Создать кастомные панели (верхнюю и нижнюю) с использованием Button, Select и иконок.

Метод: Ручная сборка.

Приоритет 2: Глобальные компоненты (Каркас сайта)
[ ] Header (Шапка сайта)

Задача: Собрать кастомный layout, включающий Логотип, Search Bar и User Menu.

Метод: Ручная сборка.

[ ] Sidebar / Filter Panel (Боковая панель)

Задача: Создать layout для боковой панели, который будет содержать в себе несколько компонентов Accordion.

Метод: Ручная сборка.

[ ] Footer (Подвал сайта)

Задача: Простой layout со ссылками и переключателем темы.

Метод: Ручная сборка.

План действий:
Установить "Готовые компоненты": Frontend-разработчик устанавливает все компоненты из первого списка с помощью CLI shadcn/ui.

Сгенерировать/собрать "Компоненты для сборки": Начать с Work Card в V0, затем последовательно собрать остальные кастомные компоненты.

Добавить все в Storybook: Каждый компонент, как готовый, так и кастомный, должен быть добавлен в Storybook с демонстрацией всех его состояний.
